01 Getting Started

Introduction

Mainspring is an on-chain witness layer for autonomous agents. It meters every decision an agent makes — the intent, the inputs it weighed, the action it took — and seals it to Casper mainnet as a permanent, independently verifiable attestation.

Where a log file can be rewritten, a Mainspring attestation cannot. It turns “trust me” into a receipt anyone — a counterparty, an auditor, a regulator — can verify without your cooperation.

Installation

Install the SDK from npm. Mainspring ships with first-class TypeScript types and works in Node 18+, Deno, and modern edge runtimes.

$ npm i mainspring

Then provision a witness key from the dashboard and set it in your environment:

MAINSPRING_KEY=msk_live_••••••••••••
MAINSPRING_NETWORK=casper-mainnet
Note

Sealing on mainnet costs a fractional amount of CSPR per attestation. Use casper-testnet during development — it's free and behaves identically.

Quickstart

Wrap any decision your agent makes. Mainspring captures it, hashes the rationale, and anchors an attestation — all without blocking your agent's main loop.

import { Mainspring } from "mainspring";

const spring = new Mainspring({ key: process.env.MAINSPRING_KEY });

// every decision has its moment — witness it
const receipt = await spring.seal({
  agent: "atlas-treasury-02",
  action: "TRANSFER 12,400 CSPR",
  rationale: "Rebalance to 30% stable reserve after weekly inflow.",
});

console.log(receipt.id);     // 0x9f3c…4a71e
console.log(receipt.block);  // #4,182,907

That's the whole loop. The returned receipt is a verifiable handle you can hand to anyone — they can confirm it on-chain without trusting you, your logs, or your uptime.

02 Core Concepts

The Loop

Every decision runs the same clock. Mainspring is built like a watch — four movements that only mean something when they turn together.

  • Act — your agent decides and acts.
  • Witness — Mainspring meters the decision at the source.
  • Anchor — the attestation is sealed to Casper.
  • Prove — anyone can replay and verify it.

Attestations

An attestation is the sealed record of one decision. The rationale is hashed (never stored in the clear), and the resulting digest is anchored on-chain alongside the agent, action, block and timestamp.

Attestation · Casper Mainnet Immutable
decision_id0x9f3c…4a71e
agentatlas-treasury-02
actionTRANSFER 12,400 CSPR
rationalesha256:b8e1…09d2 (sealed)
block#4,182,907
sealed_at2026-06-05 09:14:22 UTC

Witnesses

A witness is the lightweight hook that sits beside your agent. It is non-blocking, batches anchors efficiently, and survives restarts — if your process dies mid-decision, the witness reconciles on next boot so no decision goes unrecorded.

03 SDK Reference

capture()

Meter a decision without sealing it yet — useful when you want to enrich or review before anchoring. Returns a pending Decision you can later .seal().

const decision = spring.capture({
  agent: "scout-procure-07",
  action: "APPROVE invoice #4471",
  inputs: { amount: 4471, vendor: "verified" },
});

// …enrich, review, or gate, then:
await decision.seal("Within budget envelope; SLA met.");

seal()

Capture and anchor in one call. Resolves once the attestation is written and the block is confirmed. Pass a rationale string — it is hashed client-side before it ever leaves your process.

const receipt = await spring.seal({
  agent: "vault-keeper-01",
  action: "MINT 500 wTLS",
  rationale: "Collateral ratio 184% > 150% threshold.",
});

return receipt.url; // https://cspr.live/deploy/0x…

verify()

Independently confirm any attestation by id. This reads directly from Casper — it does not call Mainspring's servers, so it works even if Mainspring disappears tomorrow.

const result = await Mainspring.verify("0x9f3c…4a71e");

if (result.valid) {
  console.log(result.sealedAt, result.block);
}
04 Resources

Verifying on-chain

Every receipt resolves to a public deploy on cspr.live. Hand a counterparty the link, not a promise — they can inspect the sealed digest and timestamp themselves.

FAQ

Is my rationale stored on-chain?

No. Only a SHA-256 digest of the rationale is anchored. The plaintext never leaves your process unless you choose to store it yourself.

What happens if Mainspring goes offline?

Your attestations live on Casper, not on us. verify() reads the chain directly, so past records remain provable forever.

Does sealing block my agent?

No. Anchoring is asynchronous and batched. seal() can be awaited when you need the receipt, or fired and forgotten.
